# 数据库表结构

## 1. 目标

数据库需要支撑三类核心数据：

- 配置数据：用户、节点、智能体身份、房间配置
- 运行数据：消息、发言轮次、执行事件、节点会话
- 审计数据：操作日志和错误记录

首版数据库以 PostgreSQL 为准。

## 2. 设计原则

- 主键统一使用 `UUID` 或 `ULID` 字符串
- 所有时间字段使用 `timestamptz`
- Server 是唯一写库入口
- 流式输出按事件落库，最终消息再汇总回填
- 首版保留单用户部署能力，但表结构不要把自己锁死成纯单机脚本

## 3. 核心实体关系

```text
users
  -> nodes
  -> personas
  -> rooms

rooms
  -> room_members
  -> messages
  -> turns

turns
  -> turn_runs
  -> turn_events

nodes
  -> node_sessions
  -> turn_runs
```

## 4. 表清单

首版建议至少包含以下表：

- `users`
- `pair_tokens`
- `nodes`
- `node_sessions`
- `personas`
- `rooms`
- `room_members`
- `messages`
- `turns`
- `turn_runs`
- `turn_events`
- `audit_logs`

## 5. 详细表设计

### 5.1 `users`

用途：后台登录用户。即便首版只有一个管理员，也建议保留该表。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `username` | varchar(64) | 登录名，唯一 |
| `display_name` | varchar(64) | 展示名 |
| `password_hash` | varchar(255) | 密码哈希 |
| `role` | varchar(32) | 首版可固定为 `admin` |
| `status` | varchar(32) | `active` / `disabled` |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

索引建议：

- unique(`username`)

### 5.2 `pair_tokens`

用途：本地节点配对令牌。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `token` | varchar(128) | 配对令牌，唯一 |
| `created_by_user_id` | uuid | 创建人 |
| `bound_node_id` | uuid nullable | 已绑定节点 |
| `expires_at` | timestamptz | 过期时间 |
| `used_at` | timestamptz nullable | 使用时间 |
| `status` | varchar(32) | `pending` / `used` / `expired` / `revoked` |
| `created_at` | timestamptz | 创建时间 |

索引建议：

- unique(`token`)
- index(`status`, `expires_at`)

### 5.3 `nodes`

用途：代表一台安装了 `Local Agent` 的客户机器。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `name` | varchar(128) | 节点展示名 |
| `owner_user_id` | uuid | 所属用户 |
| `machine_id` | varchar(128) | 本地机器唯一标识，唯一 |
| `hostname` | varchar(128) | 主机名 |
| `platform` | varchar(32) | `darwin` / `linux` / `windows` |
| `arch` | varchar(32) | `amd64` / `arm64` |
| `agent_version` | varchar(32) | Agent 版本 |
| `status` | varchar(32) | `online` / `offline` / `disabled` / `error` |
| `last_seen_at` | timestamptz nullable | 最近心跳时间 |
| `max_workers` | integer | 最大并发 |
| `metadata` | jsonb | 预留扩展 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

索引建议：

- unique(`machine_id`)
- index(`owner_user_id`, `status`)
- index(`last_seen_at`)

### 5.4 `node_sessions`

用途：记录节点与 Server 的连接会话。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `node_id` | uuid | 所属节点 |
| `connection_id` | varchar(64) | 本次连接标识 |
| `remote_addr` | varchar(128) | 远端地址 |
| `status` | varchar(32) | `connected` / `disconnected` |
| `connected_at` | timestamptz | 建连时间 |
| `disconnected_at` | timestamptz nullable | 断开时间 |
| `disconnect_reason` | varchar(64) nullable | 断开原因 |
| `created_at` | timestamptz | 创建时间 |

索引建议：

- unique(`connection_id`)
- index(`node_id`, `connected_at`)

### 5.5 `personas`

用途：群聊里的智能体身份。首版虽然都跑 `codex`，但人格和角色配置独立。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `owner_user_id` | uuid | 所属用户 |
| `name` | varchar(64) | 身份名称 |
| `slug` | varchar(64) | 稳定标识，唯一 |
| `avatar_url` | text nullable | 头像地址 |
| `role_summary` | text | 角色说明 |
| `system_prompt` | text | 系统提示词 |
| `style_prompt` | text nullable | 风格提示词 |
| `model_provider` | varchar(32) | 首版固定 `codex` |
| `auto_reply_enabled` | boolean | 是否允许自动发言 |
| `default_timeout_sec` | integer | 默认超时 |
| `status` | varchar(32) | `active` / `disabled` |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

索引建议：

- unique(`slug`)
- index(`owner_user_id`, `status`)

### 5.6 `rooms`

用途：群聊房间。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `owner_user_id` | uuid | 创建人 |
| `name` | varchar(128) | 房间名称 |
| `topic` | text nullable | 讨论主题 |
| `mode` | varchar(32) | `mention` / `free_limited` |
| `max_rounds` | integer | 自由讨论最大轮数 |
| `max_speakers_per_round` | integer | 每轮最大发言人数 |
| `cooldown_seconds` | integer | 轮次冷却时间 |
| `status` | varchar(32) | `active` / `archived` |
| `last_message_at` | timestamptz nullable | 最近消息时间 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

索引建议：

- index(`owner_user_id`, `status`)
- index(`last_message_at`)

### 5.7 `room_members`

用途：定义房间成员，首版以智能体身份为主。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `room_id` | uuid | 所属房间 |
| `member_type` | varchar(32) | `persona` / `user` |
| `persona_id` | uuid nullable | 智能体身份 |
| `user_id` | uuid nullable | 用户 |
| `sort_order` | integer | 显示顺序 |
| `is_muted` | boolean | 是否静音 |
| `created_at` | timestamptz | 创建时间 |

约束建议：

- `member_type = persona` 时，`persona_id` 必填
- `member_type = user` 时，`user_id` 必填

索引建议：

- index(`room_id`, `sort_order`)
- unique(`room_id`, `persona_id`)

### 5.8 `messages`

用途：房间中的最终消息记录。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `room_id` | uuid | 所属房间 |
| `sender_type` | varchar(32) | `user` / `persona` / `system` |
| `sender_user_id` | uuid nullable | 发送用户 |
| `sender_persona_id` | uuid nullable | 发送身份 |
| `reply_to_message_id` | uuid nullable | 回复目标 |
| `turn_id` | uuid nullable | 关联执行轮次 |
| `content` | text | 最终消息正文 |
| `content_format` | varchar(16) | `plain` / `markdown` |
| `status` | varchar(32) | `streaming` / `completed` / `failed` / `interrupted` |
| `mentions` | jsonb | 提及列表 |
| `seq_in_room` | bigint | 房间内递增序号 |
| `created_at` | timestamptz | 创建时间 |
| `completed_at` | timestamptz nullable | 完成时间 |

索引建议：

- index(`room_id`, `seq_in_room`)
- index(`turn_id`)
- index(`created_at`)

说明：

- 一条正在流式生成的智能体消息也要先插入 `messages`
- `content` 可以先为空字符串，随着完成事件回填

### 5.9 `turns`

用途：表示一次发言调度任务，是 Server 侧的业务单元。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `room_id` | uuid | 所属房间 |
| `trigger_message_id` | uuid nullable | 触发它的消息 |
| `target_persona_id` | uuid | 目标身份 |
| `node_id` | uuid nullable | 被调度节点 |
| `trigger_type` | varchar(32) | `mention` / `free_round` / `manual` |
| `status` | varchar(32) | `queued` / `dispatched` / `running` / `completed` / `failed` / `interrupted` / `disconnected` / `cancelled` |
| `timeout_sec` | integer | 超时时间 |
| `context_size` | integer | 发送给 worker 的上下文消息数 |
| `started_at` | timestamptz nullable | 开始时间 |
| `finished_at` | timestamptz nullable | 结束时间 |
| `error_code` | varchar(64) nullable | 错误码 |
| `error_message` | text nullable | 错误说明 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

索引建议：

- index(`room_id`, `created_at`)
- index(`target_persona_id`, `status`)
- index(`node_id`, `status`)

### 5.10 `turn_runs`

用途：表示一次具体执行尝试。保留这张表，是为了支持后续重试、重连和执行排查。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | uuid | 主键 |
| `turn_id` | uuid | 所属调度任务 |
| `node_id` | uuid | 执行节点 |
| `node_session_id` | uuid nullable | 执行时的连接会话 |
| `worker_id` | varchar(64) | 本地 worker 标识 |
| `attempt_no` | integer | 第几次尝试 |
| `status` | varchar(32) | `accepted` / `running` / `completed` / `failed` / `interrupted` |
| `command_line` | text nullable | 执行命令快照 |
| `pid` | integer nullable | 本地进程号 |
| `exit_code` | integer nullable | 退出码 |
| `duration_ms` | integer nullable | 耗时 |
| `stderr_tail` | text nullable | 错误尾部日志 |
| `created_at` | timestamptz | 创建时间 |
| `started_at` | timestamptz nullable | 开始时间 |
| `finished_at` | timestamptz nullable | 结束时间 |

索引建议：

- index(`turn_id`, `attempt_no`)
- index(`node_id`, `created_at`)
- unique(`worker_id`)

### 5.11 `turn_events`

用途：保存流式事件和状态事件，支持实时展示和问题排查。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | bigserial | 主键 |
| `turn_id` | uuid | 所属调度任务 |
| `turn_run_id` | uuid nullable | 所属执行尝试 |
| `event_type` | varchar(32) | `delta` / `stderr` / `status` / `info` / `error` |
| `seq` | integer nullable | 流式顺序号 |
| `text_payload` | text nullable | 文本载荷 |
| `json_payload` | jsonb nullable | 结构化载荷 |
| `created_at` | timestamptz | 创建时间 |

索引建议：

- index(`turn_id`, `id`)
- index(`turn_id`, `event_type`)
- unique(`turn_id`, `event_type`, `seq`) where `seq is not null`

说明：

- `delta` 事件主要给前端流式拼接
- 最终仍以 `messages.content` 作为展示主文本

### 5.12 `audit_logs`

用途：记录配置变更和关键操作。

关键字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | bigserial | 主键 |
| `actor_type` | varchar(32) | `user` / `system` |
| `actor_user_id` | uuid nullable | 操作用户 |
| `action` | varchar(64) | 例如 `persona.create` |
| `resource_type` | varchar(32) | 资源类型 |
| `resource_id` | uuid nullable | 资源 ID |
| `payload` | jsonb | 详情 |
| `created_at` | timestamptz | 创建时间 |

索引建议：

- index(`actor_user_id`, `created_at`)
- index(`resource_type`, `resource_id`)

## 6. 状态枚举建议

建议在应用层先统一以下枚举：

### 节点状态

- `online`
- `offline`
- `disabled`
- `error`

### Persona 状态

- `active`
- `disabled`

### 房间模式

- `mention`
- `free_limited`

### 消息状态

- `streaming`
- `completed`
- `failed`
- `interrupted`

### Turn 状态

- `queued`
- `dispatched`
- `running`
- `completed`
- `failed`
- `interrupted`
- `disconnected`
- `cancelled`

## 7. 关键约束

- 一个 `turn` 只能指向一个目标 `persona`
- 一条智能体消息最多关联一个 `turn`
- `room_members` 必须保证同一个房间里同一个 `persona` 不重复加入
- `messages.seq_in_room` 必须在房间内单调递增
- `turn_events.seq` 只对 `delta` 和 `stderr` 类型要求严格递增

## 8. 建表顺序建议

推荐迁移顺序：

1. `users`
2. `pair_tokens`
3. `nodes`
4. `node_sessions`
5. `personas`
6. `rooms`
7. `room_members`
8. `messages`
9. `turns`
10. `turn_runs`
11. `turn_events`
12. `audit_logs`

## 9. 首版可以先不建的表

先不要为“以后可能会有”的能力提前建表：

- 多租户工作区
- 计费账单
- 文件附件
- 插件市场
- 模型适配注册表

MVP 的数据库目标是支持群聊、调度、执行和排错，不是一次性把所有未来抽象做完。
